## PART 2 — SECURITY AUDIT

### CRITICAL

#### SEC-01 — Client-Supplied Amount Bypasses DB Price Verification
- **File:** `src/app/api/paystack/initialize/route.ts:22`
- **Description:** `customAmount` from request body used directly as Paystack charge amount. A malicious POST with `{ amount: 1 }` charges GH₵ 0.01 for any order. Also: `/checkout/direct?amt=0.01` URL param trusted as invoice amount.
- **Fix:** Remove `customAmount` path entirely. Derive all amounts server-side from DB.

#### SEC-02 — `/api/abandoned/remind` Has No Authentication
- **File:** `src/app/api/abandoned/remind/route.ts`
- **Description:** No session check. Any anonymous caller can send spam emails from the store's Resend domain to arbitrary addresses.
- **Fix:** Add admin role guard identical to `/api/dispatch/route.ts`.

#### SEC-03 — `/api/email/fulfillment` Has No Authentication
- **File:** `src/app/api/email/fulfillment/route.ts`
- **Description:** No auth. Anyone can trigger fulfillment emails and SMS for arbitrary order IDs. Cost amplification attack vector.
- **Fix:** Add admin role guard.

#### SEC-04 — `/api/notify-back-in-stock` Has No Authentication
- **File:** `src/app/api/notify-back-in-stock/route.ts`
- **Description:** No auth. Caller can mark all back-in-stock requests as notified, blocking legitimate customer notifications.
- **Fix:** Add admin role guard.

#### SEC-05 — `getSession()` Used for Auth Decisions in Server Context
- **Files:** `src/app/api/me/route.ts:8`, `src/app/(shop)/checkout/success/page.tsx:114`
- **Description:** `supabase.auth.getSession()` trusts the cookie without cryptographic verification. Supabase explicitly warns against this. `getUser()` validates the JWT against the Auth server.
- **Fix:** Replace all server-side `auth.getSession()` with `auth.getUser()`.

### HIGH

#### SEC-06 — No `middleware.ts` — No Security Headers, No Edge Route Protection
- **File:** N/A (file does not exist)
- **Description:** No `Content-Security-Policy`, `X-Frame-Options`, `X-Content-Type-Options`, `Strict-Transport-Security`, `Referrer-Policy`. Dashboard routes rely on server-side redirects that run after DB queries execute. Session cookies not refreshed at edge.
- **Fix:** Create `src/middleware.ts` with security headers + Supabase session refresh + edge route guards for `/dashboard/**` and `/account/**`.

#### SEC-07 — `supabaseAdmin` (Service Role) Used for All Public Product Fetches
- **File:** `src/lib/products.ts:57`
- **Description:** Service role bypasses all RLS. Any RLS policy intended to restrict product visibility is silently ignored for shop-facing queries.
- **Fix:** Use anon-key client (`createClient()`) for public read queries. Reserve `supabaseAdmin` for privileged admin operations.

#### SEC-08 — `dangerouslySetInnerHTML` on DB-Sourced `features_list` HTML
- **File:** `src/components/ui/miss-tokyo/ProductAccordions.tsx:72`
- **Description:** Product `features_list` strings rendered directly as HTML. Admin compromise leads to stored XSS on all customer PDPs.
- **Fix:** Sanitize with `sanitize-html` server-side, or render as plain text.

#### SEC-09 — Gift Card Drain Endpoint — No Auth, No Rate Limit
- **File:** `src/app/api/gift-cards/redeem/route.ts`
- **Description:** Unauthenticated. Any caller can drain any gift card balance by guessing/knowing a valid code. Double-deduction risk with webhook handler.
- **Fix:** Restrict to internal server calls only (internal secret header), or add admin role auth.

#### SEC-10 — Cron Endpoint Unprotected When `CRON_SECRET` Not Set
- **File:** `src/app/api/cron/abandoned-cart/route.ts:19`
- **Description:** `if (process.env.CRON_SECRET && ...)` — if env var is absent, the check short-circuits and the endpoint runs unauthenticated, allowing anyone to trigger mass email sends.
- **Fix:** Invert guard: missing secret → hard 401. Never allow unauthenticated execution.

#### SEC-11 — `/api/invoice/paystack-link` Has No Authentication
- **File:** `src/app/api/invoice/paystack-link/route.ts`
- **Description:** No auth. Anyone can create Paystack payment pages using the store's account for arbitrary amounts.
- **Fix:** Add admin role guard.

#### SEC-12 — Double Inventory Deduction Race Condition
- **Files:** `src/app/api/paystack/webhook/route.ts`, `src/app/api/paystack/verify/route.ts:124`
- **Description:** Inventory decremented in both webhook and verify handlers independently. Race condition can cause double deduction depending on which arrives first.
- **Fix:** Deduct inventory only in the webhook (server-to-server, cryptographically signed). Verify endpoint updates status only.

### MEDIUM

#### SEC-13 — `dangerouslySetInnerHTML` on SVG Path Data in Footer
- **File:** `src/components/ui/miss-tokyo/Footer.tsx:175`
- **Description:** Currently hardcoded (low immediate risk), but pattern becomes XSS if social icon paths ever become admin-configurable.
- **Fix:** Use static React SVG icon components keyed by platform name.

#### SEC-14 — Two Supabase Project IDs Exposed; Second Has Wildcard Pathname
- **File:** `next.config.ts:13,19`
- **Description:** Two project hostnames visible in repo. Second entry uses `/**` pathname — too broad.
- **Fix:** Scope second entry to `/storage/v1/object/public/**`. Remove if unused.

#### SEC-15 — Account Area Protected by Client-Side Redirect Only
- **File:** `src/app/(shop)/account/layout.tsx:20`
- **Description:** Server renders and executes before client-side redirect fires. Flash of protected content. Crawlers see content.
- **Fix:** Move auth guard to middleware or convert layout to server component.

#### SEC-16 — Client-Supplied `discount_amount` Stored and Used by Webhook
- **File:** `src/app/api/paystack/initialize/route.ts:110`
- **Description:** Client-supplied `discount_amount` in metadata stored verbatim and used by webhook's `trackDiscountUsage()` to deduct gift card balance. Attacker can inflate discount to drain gift cards.
- **Fix:** Re-validate discount code and recompute amount from DB at webhook time.

#### SEC-17 — `/api/invoice/paystack-link` Trusts Client Amount
- **File:** `src/app/api/invoice/paystack-link/route.ts:15`
- **Description:** Amount for Paystack charge taken from request body, not from DB invoice record.
- **Fix:** Look up invoice amount from DB using `invoiceId`.

#### SEC-18 — HTML Injection in Order Confirmation Emails
- **File:** `src/app/api/paystack/webhook/route.ts:163`
- **Description:** `item.name`, `item.size`, `item.color` from Paystack metadata interpolated into HTML email without encoding. Allows HTML injection in customer emails.
- **Fix:** HTML-encode all user-supplied values before email template interpolation.

### LOW

#### SEC-19 — `.single()` Where `.maybeSingle()` Is Correct
- **Files:** `api/paystack/verify/route.ts:7,84`, `api/paystack/webhook/route.ts:370,573`, `api/checkout/validate-code/route.ts:16,72`
- **Fix:** Use `.maybeSingle()` for queries where zero rows is a valid outcome.

#### SEC-20 — Unvalidated `orderIds` Array Size and Format
- **Files:** `api/dispatch/route.ts:22`, `api/pickup-ready/route.ts:23`
- **Fix:** Validate max length (100) and UUID format before DB query.

#### SEC-21 — Legacy Custom Auth System (`miss_tokyo_session` Cookie)
- **File:** `src/app/api/auth/login/route.ts`
- **Description:** Parallel auth system using static `AUTH_SECRET` cookie. Redundant with Supabase Auth. Never rotates. Unclear if anything reads it.
- **Fix:** Audit consumers. If none, delete the endpoint.

#### SEC-22 — Missing Role Allowlist in `/api/invite` — Privilege Escalation
- **File:** `src/app/api/invite/route.ts:55`
- **Description:** `role` from request body passed to `inviteUserByEmail({ data: { role } })` without allowlist check. Caller can inject `role: "owner"`.
- **Fix:** Add `if (!["admin", "sales_staff"].includes(role))` guard, consistent with `/api/admin/invite-team`.

## PART 3 — PERFORMANCE AUDIT

### N+1 / Sequential Queries

#### PERF-01 — Per-Category Count Fan-Out on Homepage (HIGH)
- **File:** `src/app/(shop)/page.tsx:49`
- **Description:** N separate Supabase queries for N featured categories without `item_count_override`. Each is its own round-trip.
- **Fix:** Single `GROUP BY category_type` query, or use existing `product_count` column on `categories`.

#### PERF-02 — 4–5 Sequential Round-Trips in `getProducts()` (HIGH)
- **File:** `src/lib/products.ts:57`
- **Description:** Wholesale category fetch, category slug resolve, main products query, price bounds query, category name map — all sequential. Steps 1 and 5 both query `categories` independently.
- **Fix:** Combine steps 1+5 into one query. Run price bounds, category resolution, and main query in `Promise.all`. Cache wholesale category list with `unstable_cache`.

#### PERF-03 — Sequential `categories` Round-Trip After Every PDP Load (HIGH)
- **File:** `src/lib/products.ts:226`
- **Description:** Full `categories` table fetched on every `getProductBySlug` call just to resolve one `category_type → slug` mapping.
- **Fix:** Join the category in the product query, or cache categories at module level.

#### PERF-04 — Price Bounds Fetches All Product Prices Into Memory (HIGH)
- **File:** `src/lib/products.ts:138`
- **Description:** Full `price_ghs` column for all active products fetched with no limit, then min/max computed in JavaScript.
- **Fix:** Two parallel `.limit(1)` queries (ascending + descending), or a Postgres aggregate.

#### PERF-05 — `getVideoProducts` Fetches Entire Catalog, Filters in JS (MEDIUM)
- **File:** `src/lib/products.ts:313`
- **Description:** All active products loaded, then `.mp4`/`.mov` filter applied in JavaScript. No limit.
- **Fix:** Add DB-level filter on `video_url` + `.limit(50)`.

## PART 5 — LOGIC / DEAD CODE AUDIT

### Critical Logic Bugs

#### LOG-01 — Silent Swallow of Order Insert Error — Payment Still Proceeds (HIGH)

- **File:** `src/store/useCart.ts:53`, `src/app/api/paystack/initialize/route.ts:59`

#### LOG-04 — Missing Cleanup for `setTimeout` in `VideoCard` and `ProductOptions` (HIGH)

#### LOG-06 — Price Debounce Timer Leaks on Unmount (LOW)
